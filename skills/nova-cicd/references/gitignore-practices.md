# .gitignore Best Practices

Standard `.gitignore` rules for Nova projects. Apply these when setting up a new repo or auditing an existing one.

## Base Template (Node/Next.js/Convex)

```gitignore
# Dependencies
node_modules/

# Build output
.next/
out/
dist/
build/

# Environment files — NEVER commit these
.env
.env.local
.env.development.local
.env.test.local
.env.production.local
.env*.local

# Convex
.convex/

# Vercel
.vercel/

# OS files
.DS_Store
Thumbs.db

# IDE
.idea/
.vscode/
*.swp
*.swo

# Debug logs
npm-debug.log*
yarn-debug.log*
yarn-error.log*

# TypeScript cache
*.tsbuildinfo

# Test coverage
coverage/

# Misc
*.pem
```

## Critical Rules

### Never commit secrets
- `.env` files with API keys, deploy keys, database URLs
- `*.pem` files (SSL certificates, private keys)
- Convex deploy keys (`CONVEX_DEPLOY_KEY`)
- Any file containing `sk_`, `secret_`, or `PRIVATE_KEY`

### Never commit build artifacts
- `node_modules/` — always install from lockfile
- `.next/`, `dist/`, `build/` — regenerated on deploy
- `.convex/` — generated by Convex CLI

### Always commit lockfiles
- `package-lock.json` or `yarn.lock` — ensures deterministic installs
- Do NOT gitignore these

### OS/IDE files
- `.DS_Store` (macOS) and `Thumbs.db` (Windows) should always be ignored
- IDE config (`.vscode/`, `.idea/`) — ignore unless the team agrees to share settings

## Verifying .gitignore Coverage

```bash
# Check if any tracked files should be ignored
git ls-files --cached --ignored --exclude-standard

# Remove a file that was already tracked before being added to .gitignore
git rm --cached <file>
git commit -m "chore: remove tracked file that should be ignored"
```

## When to Check

- **New repo setup**: Ensure `.gitignore` is created from this template
- **Adding new tools/frameworks**: Add their output/cache dirs to `.gitignore`
- **Before first commit**: Verify no secrets or build artifacts are staged
- **PR review**: Flag any `.env`, `node_modules/`, or build dirs that shouldn't be committed
